# Contributing to ShitBucket

First off, thank you for considering contributing to ShitBucket! It's people like you who make it a better tool for everyone.

## How Can I Contribute?

### Reporting Bugs
- Use the [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md).
- Provide a clear, descriptive title.
- Describe the exact steps to reproduce the issue.

### Suggesting Enhancements
- Use the [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md).
- Explain why this feature would be useful to most users.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the existing style (Brutalist, high-contrast).
5. Open a PR with a clear description of your changes.

## Design Philosophy
ShitBucket follows a **Brutalist** aesthetic:
- **Hard Shadows:** Use the `.shadow-hard` utilities.
- **High Contrast:** Cream background (`#FFF8EE`), black borders, orange accents.
- **Mobile First:** Ensure everything feels "physical" on a touch screen.

## Questions?
Feel free to open an issue with the tag `question`.
